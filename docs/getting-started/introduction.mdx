## 프론트엔드 모니터링

웹 프론트엔드 성능 모니터링은 사용자 경험을 개선하고 웹 애플리케이션의 성능을 최적화하는 데 중요한 역할을 합니다. IMQA를 사용하면 표준화된 방식으로 성능 데이터를 수집하고 분석할 수 있습니다.

수집된 데이터를 바탕으로 다음과 같은 최적화를 수행할 수 있습니다:

- 느린 API 호출 식별 및 최적화
- 클라이언트 사이드 렌더링 성능 개선
- 자주 발생하는 오류 해결

## 계측 항목

- XMLHttpRequest 및 Fetch API 자동 계측
- 사용자 상호작용 (클릭, submit, 드래그 등)
- Document load, Fetched resources
- History API 및 해시 변경 지원
- 웹 바이탈(Web Vitals)
- 세션 ID
- 자동 컨텍스트 연결을 통한 longtasks
- uncaught exceptions, unhandled rejections, document errors, console errors
- 수동 계측 지원
- Timer, Promise, Native async-await, 이벤트, 옵저버 등을 통한 자동 컨텍스트 유지
