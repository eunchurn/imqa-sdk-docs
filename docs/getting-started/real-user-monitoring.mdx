### RUM이란 무엇인가요?

RUM(Real User Monitoring)은 여러분의 웹사이트나 앱을 실제 사용자들이 어떻게 사용하고 있는지 생생하게 보여주는 강력한 도구입니다. 마치 사용자의 어깨 너머로 화면을 함께 보는 것처럼, 그들의 경험을 직접 체험할 수 있게 해줍니다.
RUM으로 무엇을 할 수 있나요?

- 사용자 여정 추적: 클릭부터 페이지 로딩까지, 사용자의 모든 행동을 실시간으로 확인할 수 있습니다.
- 성능 병목 발견: 어떤 부분에서 사용자들이 기다리고 있는지, 어떤 기능이 느린지 정확히 파악할 수 있습니다.
- 오류 즉시 포착: 사용자가 경험하는 모든 오류를 실시간으로 감지하고 기록합니다.
- 사용자 그룹 분석: 지역, 브라우저, 기기별로 사용자 경험이 어떻게 다른지 비교할 수 있습니다.

### IMQA의 RUM은 어떻게 작동하나요?

- 간편한 설치: IMQA의 JavaScript 라이브러리를 여러분의 웹사이트나 앱에 추가하기만 하면 됩니다.
- 자동 데이터 수집: 사용자의 모든 상호작용을 자동으로 수집합니다. 특별한 코드 작성이 필요 없습니다.
- 실시간 분석: 수집된 데이터를 즉시 분석하여 의미 있는 인사이트를 제공합니다.
- 호환성: 업계 표준인 OpenTelemetry와 완벽히 호환되어, 다른 도구들과의 연동도 쉽습니다.

### 왜 IMQA의 RUM을 선택해야 할까요?

- 완벽한 투명성: 오픈소스 기반으로, 여러분의 데이터가 어떻게 처리되는지 모든 것을 확인할 수 있습니다.
- 유연한 설정: 여러분의 비즈니스 요구에 맞춰 자유롭게 설정을 조정할 수 있습니다.
- 포괄적인 지원: 단순한 웹사이트부터 복잡한 단일 페이지 애플리케이션(SPA)까지 모두 지원합니다.


IMQA의 RUM으로 여러분의 디지털 제품을 한 단계 더 발전시켜보세요. 사용자들이 정말로 원하는 것이 무엇인지, 그들의 눈으로 직접 확인할 수 있습니다.

더 자세한 설정 방법과 고급 기능은 [IMQA OpenTelemetry JavaScript 설정 가이드](../web-agent/installation.mdx)에서 확인하실 수 있습니다.

## 계측 항목

- XMLHttpRequest 및 Fetch API 자동 계측
- 사용자 상호작용 (클릭, submit, 드래그 등)
- Document load, Fetched resources
- History API 및 해시 변경 지원
- 웹 바이탈(Web Vitals)
- 세션 ID
- 자동 컨텍스트 연결을 통한 longtasks
- uncaught exceptions, unhandled rejections, document errors, console errors
- 수동 계측 지원
- Timer, Promise, Native async-await, 이벤트, 옵저버 등을 통한 자동 컨텍스트 유지
